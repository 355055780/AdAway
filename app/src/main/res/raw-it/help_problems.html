Spazio insufficiente nella partizione

Prova a cambiare il percorso del file hosts nelle preferenze in /data/data/hosts (o /data/hosts) e applica di nuovo AdAway.

Non funziona in Android 4.4+

Prova a cambiare il percorso del file hosts nelle preferenze da /data/data/hosts a /data/hosts o /system/etc/hosts e applica di nuovo AdAway.

Non blocca le pubblicità nell'applicazione XYZ!

Alcuni hostname potrebbero mancare nei file hosts forniti in Sorgenti Host oppure l'applicazione contiene al suo interno le immagini per mostrare pubblicità senza collegarsi a internet.

Puoi registrare le richieste DNS (Menu->Registra Richieste DNS) da AdAway per scoprire quali ulteriori hostname devono essere bloccati.

Aggiungi gli hostname sospetti alla tua personale Blacklist tenendo premuta la voce nel Registro e quando li hai verificati riportali nel Forum Hosts Inbox of hosts-file.net.

Pubblicità conservata nella cache

Alcune applicazioni conservano le pubblicità nella cache dopo il download. Questo fa sì che la pubblicità rimanga in alcune applicazioni. Puoi provare a eliminare la cache di queste applicazioni nella Lista delle Applicazioni Android per risolvere questo problema.

L'applicazione XYZ non funziona più!

Alcune applicazioni hanno bisogno di comunicare con un host bloccato da AdAway o non funzionano quando gli host che dovrebbero generare la pubblicità non sono raggiungibili. Vai su https://github.com/dschuermann/ad-away/wiki/ProblematicApps per ottenere una lista delle applicazioni conosciute che mostrano problemi. In alternativa, cerca i nomi degli host necessari e aggiungili alla tua Whitelist sotto Le Tue Liste e segnalali al tracciatore di bug di AdAway.

Il pulsante indietro nei browser non funziona

Puoi abilitare il webserver locale di AdAway nelle preferenze per aggirare il problema
